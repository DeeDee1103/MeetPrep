## Authentication

All **protected** endpoints require a valid Supabase session (set via cookie on
browser requests, or `Authorization: Bearer <token>` for server-to-server calls).

**Webhook endpoints** use their own signature verification:

| Webhook | Verification header |
|---------|---------------------|
| Google Calendar | `x-goog-channel-token` (matches userId stored at watch setup) |
| Stripe | `stripe-signature` (HMAC using `STRIPE_WEBHOOK_SECRET`) |

## Error responses

All endpoints return JSON. On error:

```json
{ "error": "Human-readable message" }
```

| Status | Meaning |
|--------|---------|
| `400` | Bad request / missing parameters |
| `401` | Not authenticated |
| `402` | Plan limit reached — upgrade required |
| `404` | Resource not found |
| `409` | Conflict (e.g. brief already generating) |
| `500` | Internal server error |
